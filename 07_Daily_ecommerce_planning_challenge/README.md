# 100일 이커머스 기획 챌린지

## 목적

매일 하나의 이커머스 아이디어를 기획하여
실무 감각을 키우고 구조적 사고력을 강화하며
최종적으로 포트폴리오를 완성합니다.

---

## 폴더 구조

```
07_Daily_ecommerce_planning_challenge/
├── 1_Daily_Records/              # 일별 기획 파일 (day01 ~ day100)
├── 2_30day_Summary/              # 30일 회고 (30일, 60일, 90일)
├── 3_Idea_Pool/                  # 아이디어 풀 (ideas.txt, used_ideas.txt)
├── 4_Framework_Guide/            # 프레임워크 정리 문서
├── 5_AI_Prompt_Guide/            # 일일 기획용 AI 프롬프트 가이드
├── 6_Portfolio_Drafts/           # 포트폴리오 초안 (베스트 기획 모음)
├── 7_Final_Portfolio/            # 최종 포트폴리오 (Top 10 정리)
└── run_today.sh                  # 오늘의 챌린지 자동 실행 스크립트
```

---

## 사용법

### 1. 오늘의 챌린지 시작

```bash
bash run_today.sh
```

이 스크립트는 다음을 수행합니다:

* `3_Idea_Pool/ideas.txt`에서 사용하지 않은 랜덤 아이디어 하나를 선택
* `1_Daily_Records/dayXX/README.md` 생성
* 사용한 아이디어를 `used_ideas.txt`에 기록
* 남은 아이디어 개수 표시

---

### 2. 기획 작성

생성된 `dayXX/README.md` 파일을 열어 7가지 항목을 작성합니다:

1. 한 줄 요약
2. 문제 발생 시점
3. 기존 해결 방식과 불편함
4. 핵심 가설
5. 최소 화면 1개 설명
6. 가장 위험한 지점
7. 오늘의 기획자 인사이트

#### 15분 기획 원칙

완성된 사업 설계가 아니라, 핵심 가설을 빠르게 구조화하는 것이 목표입니다.
속도를 제한해야 본질만 남고, 반복을 통해 사고 근육이 강화됩니다.

#### 일별 기획 프레임워크 적용 (순환 사용)

일별 기획 시 아래 프레임워크를 돌아가면서 사용합니다.

* SCAMPER
* JTBD
* Lean Canvas
* AARRR 관점 체크

권장 방식:

* 기본: JTBD + Lean Canvas
* 아이디어 확장 필요 시: SCAMPER
* 작성 후 점검: AARRR 퍼널 위치 명시

목표:
빠른 사고력과 문제 구조화 훈련

---

### 3. AI 검토 및 복기 루틴

일일 기획은 다음 30분 구조로 운영합니다.

#### 1단계. 기획 작성 (15분)

7가지 항목을 기반으로 가설 중심 기획 작성

#### 2단계. AI 반박 (5분)

AI를 활용하여:

* 실패 가능성 5가지 도출
* 가장 위험한 가설 1개 지정
* 최소 비용 검증 방법 제안

목적:
확신을 깨고 리스크를 먼저 드러내기

#### 3단계. 복기 (5분)

AI 답변을 바탕으로 기록합니다:

* 내가 놓친 부분
* 가설 수정 여부
* 오늘 배운 점 1줄

목적:
사고를 정교화하고 학습을 축적

#### 4단계. 여유 시간 (5분)

필요 시 확장 분석, 문장 정제, 슬로건 작성 등 수행

총 30분 이내 완료

---

### 4. 30일 회고

매 30일마다 `2_30day_Summary` 폴더에 회고를 기록합니다.

#### 30일 회고 프레임워크 (순환 적용)

* KPT
* SWOT
* Five Forces
* 기획 주제 패턴 분석

권장 방식:

* 기본 회고: KPT
* 30일 중 베스트 아이디어 1개에 SWOT 또는 Five Forces 적용
* 다룬 문제 영역, 타겟, 퍼널 위치를 분류하여 패턴 분석

목표:
전략적 시야 확보 및 사고 확장

---

### 5. 포트폴리오 준비 (80일차 이후)

80일 이후부터 포트폴리오 정리를 시작합니다.

* 베스트 기획 10개 선정
* `6_Portfolio_Drafts/`에서 고도화
* 최종본을 `7_Final_Portfolio/`에 정리

#### 포트폴리오 정리 프레임워크 적용

* Business Model Canvas (BMC)
* GTM 전략 추가
* Value Proposition Canvas

권장 방식:

* 각 아이디어에 BMC 적용
* 타겟 고객과 가치 제안 명확화
* GTM 전략과 수익 구조 정리
* 고객 Pain/Gain과 해결 구조 연결

목표:
아이디어 나열이 아닌 사업 설계 능력 증명

---

## 챌린지 규칙

1. 하루도 빠짐없이 – 100일 연속 기록
2. 15분 안에 완성 – 빠른 사고력 훈련
3. 실제 문제 중심 – 가상이 아닌 진짜 불편함 해결
4. 중복 없이 진행 – `used_ideas.txt`로 자동 관리
